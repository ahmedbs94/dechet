Étapes de configuration

1

Allez sur Facebook Developer Console

Connectez-vous avec votre compte Facebook et accédez à votre application TriDéchet.

🔗 Ouvrir Facebook Developer Console

2

Ajouter le Key Hash Android

Dans la section des paramètres basiques, trouvez la plateforme Android et ajoutez le Key Hash :

  1. Faites défiler vers "Ajouter une plateforme" → choisissez Android
  2. Nom du package : com.example.tri_dechet
  3. Classe d'activité par défaut : com.example.tri_dechet.MainActivity
  4. Dans le champ "Key Hashes", collez : uCWfYSjWNPi+Q7I258CCTn5zSBU=
  5. Cliquez sur Enregistrer
⚠️
CRITIQUE : Sans ce Key Hash, Facebook bloquera toutes les tentatives de connexion avec l'erreur "Invalid key hash". C'est la cause #1 d'échec de l'auth Facebook sur Android.
3

Activer le Login Facebook

Allez dans Produits → Facebook Login → Paramètres et vérifiez :

  • Client OAuth Login activé
  • Web OAuth Login activé
  • OAuth Login Android natif → Activé
  • Forcer le HTTPS → Activé
🔗 Paramètres Facebook Login
4

Vérifier les testeurs de l'application

Si votre application est en mode développement (non publiée), seuls les comptes Facebook listés comme Testeurs peuvent se connecter.

🔗 Gérer les testeurs
Alternative rapide : Allez dans Paramètres → Avancés → Mode développement et basculez en mode "Live" pour permettre à n'importe quel compte Facebook de se connecter. Mais assurez-vous que votre app est conforme aux directives Facebook avant de faire ça.
5

Backend : Démarrer le serveur

Le backend FastAPI doit être actif pour valider les tokens Facebook :

cd backend
venv\Scripts\activate
python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000

Ou plus simplement, double-cliquez sur : scripts\dev.bat

Sur appareil physique Android : Le serveur doit écouter sur 0.0.0.0 (pas seulement localhost) et votre téléphone doit être sur le même réseau Wi-Fi que votre PC.

L'IP locale de votre PC est : 192.168.1.13 (configurée automatiquement dans constants.dart)

🔧 Erreurs courantes et solutions

❌ "Invalid key hash"

Le Key Hash n'est pas configuré ou est incorrect. Recommencez l'étape 2.

❌ "Connexion annulée" immédiatement

Ça arrive si le dialog Facebook s'ouvre puis se ferme aussitôt. Causes possibles :

  • La permission email n'est pas disponible (compte Facebook sans email)
  • L'App ID ou le Client Token est incorrect dans strings.xml
  • Le Key Hash est invalide

❌ "Serveur inaccessible"

Le backend Python n'est pas démarré. Lancez scripts\dev.bat.

Sur appareil physique : vérifiez que vous êtes sur le même Wi-Fi et que le firewall autorise le port 8000.

❌ "User not authorized" ou "Access denied"

L'application Facebook est en mode développement et votre compte n'est pas dans la liste des testeurs (étape 4).
